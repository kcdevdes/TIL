## 연산자

값에 대해서 어떤 작업을 컴퓨터에게 지시하기 위한 기호이다.

```js
a = 1 // =은 대입 연산자이다.
```



## 비교 연산자

프로그래밍에서 주어진 값들이 서로 같은지, 다른지, 큰지, 작은지를 구분하는 연산자를 비교 연산자라 한다. 이때의 결과는 true 혹은 false로 모이며, true와 false는 boolean(불린)이라고 불리는 데이터 형식이다.

```js
// ==연산자 (동등 연산자)
alert ( 1 == 2 ) // false
alert ( 2 == 2 ) // true
alert ( "wor" == "war" ) // false
alert ( "one" == "one" ) // true

// === 연산자 (일치 연산자)
alert (1 == '1') // true
alert (1 === '1') // false
```

위의 === 연산자에서는 1과 '1'를 서로 다른 값으로 인식하였다. '정확'의 의미를 따지는 연산자로 서로 같은 수를 표현하고 있더라고 데이터 형이 같은 경우만 true로 판단하는 연산자인 것이다. 결론은 ==보단 ===이 매우 추천된다.

```js
alert ( null == undefined ); // true
alert ( null === undefined ); // false
alert ( true == 1 ); // true
alert ( true === 1 ); // false
alert ( true == '1' ); // true
alert ( true === '1'); // false

alert ( 0 === -0 ); // true
alert ( NaN === NaN ); // false
```

null과 undefined는 값이 없다는 의미이다. null은 **값이 없음** 을 명시적으로 표시한 것이고, undefined는 값 자체가 존재하지 않는 상태인 것이다. NaN은 0/0 (Divide by zero)와 같은 연산의 결과로 만들어지는 특수한 데이터 형인데 숫자가 아니라는 뜻이다.

## 비교 연산자

* **!= **
  * 같지 않음을 나타내는 의미로 ==와는 정반대의 결과를 보여준다.
* **!==**
  * 정확하게 같지 않다는 의미이다.
* **>**
  * 좌항이 우항보다 크다면 참, 그렇지 않다면 거짓임을 알려주는 연산자다. 
  * 반대로는 **<** 연산자가 있다.
